# Decision DEC-0003

Status: Accepted

## Context

`OpenAiService.classify()` uses OpenAI SDK v3 with free-text prompts and manual
JSON parsing. OpenAI now recommends Structured Outputs (JSON Schema, `strict:
true`) for classification tasks with enum-constrained categories (per R-0002).

## Options

1. **Migrate now** — upgrade SDK, adopt Structured Outputs with category enum schema
2. **Defer** — keep current prompt+parse approach; migrate when adding tests or SDK breaks
3. **Fine-tune** — train custom model on user's labelled transactions (OpenAI cookbook pattern)

## Recommendation

**Option 2: Defer migration.**

Current approach works in production. Migration requires SDK upgrade (v3 → v4+),
prompt rewrite, and regression testing — blocked until test harness exists (R6).

## Impact

- **Positive:** No breaking change risk now; pre-AI layers already reduce invalid-category failures.
- **Negative:** Occasional parse failures / retry overhead remain; tech debt tracked as R9.
- **Revisit trigger:** When `tests/run-tests.ps1` (or equivalent) exists and covers `OpenAiService`.
