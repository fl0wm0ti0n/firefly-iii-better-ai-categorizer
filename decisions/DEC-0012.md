# Decision DEC-0012

Status: Accepted

## Context

US-0004 AC-1 requires a configurable dominance threshold (default 80%). The
threshold determines when account history produces a suggestion: if the dominant
category share ≥ threshold, a history suggestion is generated with
confidence = dominance share (AC-2).

Additionally, a minimum transaction count prevents statistically weak dominance
from small samples (e.g., 4/5 = 80% but only 5 transactions).

## Options

1. **Default 80% threshold + 10 min transactions** — configurable via
   `data/dominance-config.json`; conservative defaults
2. **Default 80% threshold + 5 min transactions** — lower bar for suggestions;
   more accounts qualify but weaker signal
3. **Per-account threshold** — configurable per expense account; flexible but
   complex UI and storage
4. **Hardcoded 80%** — no configurability; simple but inflexible

## Recommendation

**Option 1: Default 80% threshold + 10 min transactions, configurable via
`data/dominance-config.json`.**

Rationale:
- AC-1 explicitly requires configurability ("configurable via UI or `data/` config")
- 80% is conservative — reduces false positives (accounts with mixed spending)
- 10 min transactions provides statistical significance for personal finance
  patterns (per R-0020, industry references: Midday uses 5, Expense Atlas uses 10)
- File-based config follows existing pattern (`auto-categorization-config.json`)
- UI configuration deferred to US-0005 (admin UI consolidation)

Trade-offs:
- 10 min transactions means new/small accounts never get suggestions (intentional)
- No per-account override in v1 (future enhancement if operator requests)
- Config file requires manual edit or future API endpoint to change

## Configuration schema

```json
{
  "dominanceThreshold": 0.80,
  "minTransactionCount": 10
}
```

## Impact

- **Positive:** Conservative defaults prevent false positives; configurable for
  operator tuning
- **Negative:** Small accounts (< 10 categorized tx) never get suggestions
- **Linked:** R-0020, US-0004 (AC-1, AC-2), architecture § US-0004
