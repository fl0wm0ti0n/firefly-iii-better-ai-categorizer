# Decision DEC-0001

Status: Accepted

## Context

The categorizer supports three entry points — Firefly webhook, bulk processing
API, and test webhook — that must produce identical categorization behavior.
Operators configure rules (account mappings, auto-cat, word/keyword mappings)
expecting consistent results regardless of trigger.

## Options

1. **Unified pipeline** — single `#resolveCategory()` method called by all paths
2. **Per-path logic** — duplicate or divergent pipelines per entry point
3. **Firefly native rules only** — rely on Firefly III `/api/v1/rules`, drop AI layers

## Recommendation

**Option 1: Unified pipeline.**

One `#resolveCategory(transaction, categories)` in `App.js` with fixed precedence:
account mapping → auto-cat → word mappings → keyword hints → OpenAI.

## Impact

- **Positive:** Predictable behavior; README pipeline diagram matches code; bulk
  and webhook always agree.
- **Negative:** All pipeline changes touch `App.js` (monolith risk R1).
- **Linked:** R-0001, architecture Components §2.
