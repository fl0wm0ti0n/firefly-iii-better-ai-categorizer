# Decision DEC-0014

Status: Accepted

## Context

US-0003 migrated `OpenAiService.classify()` to Structured Outputs with schema
`{ category: string }` (plus `UNKNOWN`). US-0004 AC-3 requires AI confidence
in the classification result for display in the review queue (AC-5) and
comparison with history dominance (AC-4).

The current `classify()` return shape is `{ category, prompt, response }`.
Adding `confidence` changes the return contract.

## Options

1. **Extend schema with `confidence` field** — add `confidence: { type: 'number',
   minimum: 0, maximum: 1 }` to existing `json_schema`; return
   `{ category, confidence, prompt, response }`; update tests
2. **Separate method `classifyWithConfidence()`** — new method with extended
   schema; `classify()` unchanged; dual API surface
3. **Parse confidence from response text** — remove strict mode; prompt for
   `{ category, confidence }`; manual JSON parse
4. **Heuristic confidence from logprobs** — use `logprobs` API parameter;
   calculate confidence from token probabilities

## Recommendation

**Option 1: Extend existing `classify()` schema with `confidence` field.**

Rationale:
- Simplicity: single method, one schema, one return shape
- US-0003 strict mode is complete; extend rather than duplicate
- `App.js` `#resolveCategory` ignores extra fields — no breaking change to
  pipeline callers (JavaScript ignores extra object properties)
- Test impact: update 5 test cases in `openAiService.test.js` + 4 stub returns
  in `resolveCategory.test.js` to include `confidence` field
- Prompt calibration: guide model with confidence ranges (0.9–1.0 clear,
  0.6–0.8 moderate, 0.3–0.5 uncertain)

Trade-offs:
- Model confidence is heuristic (not calibrated); document for operator
- Existing test stubs must update to include `confidence` field
- `gpt-3.5-turbo` may not support confidence well (already unsupported for
  Structured Outputs per US-0003)

Schema extension:
```json
{
  "type": "object",
  "properties": {
    "category": { "type": "string", "enum": ["...categories", "UNKNOWN"] },
    "confidence": { "type": "number", "minimum": 0, "maximum": 1 }
  },
  "required": ["category", "confidence"],
  "additionalProperties": false
}
```

Return shape:
```js
{ category: string | null, confidence: number, prompt: string, response: string }
```

## Impact

- **Positive:** Single method; preserves strict mode; minimal caller change
- **Negative:** Test stub updates required; confidence is heuristic not calibrated
- **Linked:** R-0019, US-0004 (AC-3, AC-5), US-0003 (AC-2), architecture § US-0004
