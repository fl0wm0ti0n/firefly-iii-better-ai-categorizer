# Decision DEC-0002

Status: Accepted

## Context

The repo ships with a local `docker-compose.yml` (standalone-dev profile, port
3001) marked DEPRECATED. Production runs from the parent Firefly stack at
`/workdir/firefly/docker-compose.yml` as service `categorizer` on port 3000
behind Traefik (`categorizer.omniflow.cc`).

## Options

1. **Parent stack (canonical)** — build context points to this repo; orchestrated
   with Firefly III, shared network, Traefik labels
2. **Repo-local compose as primary** — self-contained deployment from this repo
3. **Kubernetes / cloud-native** — separate orchestration layer

## Recommendation

**Option 1: Parent stack for production.**

Repo-local compose remains for optional standalone-dev testing only. Documentation
and operator runbooks reference parent stack commands.

## Impact

- **Positive:** Single production path; co-located with Firefly; Traefik TLS.
- **Negative:** Developers must know parent stack location; repo compose is confusing if not clearly marked deprecated (already done in file header).
- **Linked:** architecture Overview, codebase-map Deployment topology.
