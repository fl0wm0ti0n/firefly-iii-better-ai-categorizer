## Status

Accepted (2026-06-23)

## Context

BUG-0002 reported `GET /api/reviews` returning HTTP 404 HTML and `loadPendingReviews` throwing a JSON parse error. Research (R-0024) confirmed root cause H1: the production container image `firefly-categorizer:latest` predates US-0004, so the `/api/reviews` route is not present at runtime. The source code already contains the route and handler.

## Decision

Adopt a three-tier fix:

1. **Primary (required):** Rebuild and redeploy the production container from the parent stack using `--build --force-recreate`.
2. **Secondary (strongly recommended):** Defensive UI hardening in `public/index.html` `loadPendingReviews` â€” check `response.ok` and guard the response content-type before calling `.json()`, then surface a structured error.
3. **Tertiary (recommended follow-up):** Add an operational health/version probe and a runbook rule to prevent stale-image regressions.

No backend code changes are required; the route already exists in source.

## Options evaluated

| Option | Verdict | Rationale |
|---|---|---|
| Rebuild/redeploy only | Insufficient | Fixes the current symptom but leaves the UI brittle to transient failures. |
| UI hardening only | Insufficient | Does not restore `/api/reviews`; the operator still sees an empty panel and an error toast. |
| Add backend fallback / duplicate route | Rejected | The route already exists; adding a fallback adds code debt without fixing deployment hygiene. |
| Move `/api/reviews` to a separate microservice | Rejected | Massive overkill for a stale-image problem. |
| Three-tier fix (chosen) | Accepted | Repairs deployment, hardens UX, and begins prevention. |

## Tradeoffs

- `--build --force-recreate` causes a brief container restart (seconds), which is acceptable for a bug-fix deploy.
- UI hardening adds small complexity to `loadPendingReviews`, but aligns with BUG-0002 AC-3 and the BUG-0001 defensive pattern.
- The live `public/` mount means the UI fix can be rolled out independently of an image rebuild, enabling faster rollback and forward fixes.
- The version probe (tertiary) is deferred to a future quick/technical-debt item to keep BUG-0002 scope tight.

## Rollback plan

If redeploy causes regression:

1. Stop the new container from the parent stack: `docker compose stop categorizer`.
2. Re-run with the previous image if available (`docker compose up -d categorizer` with an explicit prior tag), or restore from a host image backup.
3. Revert the UI hardening via git checkout of `public/index.html` â€” the live mount means it takes effect immediately.
4. Verify rollback with the same internal probe: `http://127.0.0.1:3000/api/reviews` should return the pre-redeploy behavior.

## Acceptance closure criteria

- Production internal probe returns HTTP 200 JSON from `/api/reviews`.
- Browser console shows no `SyntaxError` on authenticated page load.
- `loadPendingReviews` surfaces an error toast if the endpoint returns a structured `{success:false,error:...}` response.
