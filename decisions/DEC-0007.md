# Decision DEC-0007

Status: Accepted

## Context

US-0002 AC-2 requires `python scripts/validate_readme_feature_coverage.py --report`
to exit 0. Research R-0010 confirmed this brownfield Firefly III fork lacks:

- A `template/` tree (template parity sub-check fails on three path pairs)
- An `installer` Python module (script entry fails at import)

The repo is a **consumer fork**, not a full its-magic kit install. Runbook already
documents `--no-template-parity` for `validate_doc_profile.py` on fixture/brownfield
trees.

## Options

1. **Brownfield exempt + `--no-template-parity`** — skip template byte-parity;
   vendor minimal `scripts/installer.py` for scratchpad merge so canonical script runs.
2. **Materialize full `template/` mirror** — satisfy parity without flag; dual-edit
   all README coverage scripts.
3. **Lib-direct audit only** — `PYTHONPATH=scripts` invoke `readme_feature_coverage_lib`;
   does not satisfy AC-2 literal script command.
4. **Disable `README_FEATURE_COVERAGE_ENFORCE`** — reject; scratchpad enforce=1 is active.

## Recommendation

**Option 1** for US-0002 and this repo's ongoing docs gates.

- All blocking validator invocations (local, CI, `/release` 3f) pass
  `--no-template-parity` until a deliberate kit upgrade materializes `template/`.
- Execute vendors minimal `scripts/installer.py` with `merge_scratchpad_layers` only
  (DEC-0055 contract) so AC-2 canonical command works without parent kit checkout.
- Update runbook canonical commands to include the flag (mirror `validate_doc_profile`
  brownfield note).

Option 2 deferred — high maintenance, no operator value for this fork.
Option 3 insufficient for AC-2 acceptance literal.
Option 4 rejected — leaves release gate red.

## Impact

- **Positive:** Unblocks AC-2 without spurious parity FAIL; aligns with existing
  brownfield runbook pattern; lib-direct audit remains available for sprint dev.
- **Negative:** Template drift vs kit is not enforced in this repo until `template/`
  exists; installer stub must stay compatible with scratchpad merge semantics.
- **Execute gates:** Architecture should schedule installer vendoring before enforce
  validation task; README coverage lines per R-0013 before US-0002 DONE.
