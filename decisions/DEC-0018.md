# DEC-0018 — Standalone docker-compose.local.yml for agent-driven UAT

## Status

Accepted (2026-06-22)

## Context

US-0006 requires an ephemeral local launch seam so the Cursor IDE agent can start the Categorizer container, poll health, and run browser UAT probes. The repo has multiple compose files:

- `docker-compose.yml` — deprecated standalone-dev (profile `standalone-dev`, port 3001:3000, healthcheck probes `GET /health`)
- `docker-compose.dev.yml` — hot-reload dev (port 3000:3000, healthcheck probes `/health`)
- Parent stack `/workdir/firefly/docker-compose.yml` — production (port 3000, Traefik → `categorizer.omniflow.cc`)

Research R-0023 Q5 identified the risk of `docker-compose.override.yml` auto-merge: implicit merge could contaminate parent stack or dev compose.

## Decision

Create **`docker-compose.local.yml`** as a standalone file invoked with explicit `-f` flag:

```bash
docker compose -f docker-compose.local.yml up -d
docker compose -f docker-compose.local.yml down
```

**Rationale:**

1. **No implicit merge** — explicit `-f` prevents cross-contamination with parent stack or dev compose.
2. **Ephemeral lifecycle** — agent controls start/stop; no profile friction (profiles add `--profile` flag complexity).
3. **Isolated healthcheck** — probe `GET /` (matches Dockerfile + App.js), not `/health` (which doesn't exist).
4. **Port isolation** — host 3001 → container 3000; no conflict with parent (3000) or dev (3000).
5. **No Traefik labels** — direct port binding; Cursor browser MCP reaches `localhost:3001` without restriction.

## Alternatives considered

| Alternative | Rejected because |
|-------------|------------------|
| `docker-compose.override.yml` | Auto-merge risk; implicit contamination of parent/dev stack |
| Reuse `docker-compose.yml` (deprecated) | Profile `standalone-dev` requires `--profile` flag; healthcheck probes `/health` (wrong) |
| Add `/health` route to App.js | scope_out prohibits app logic changes; `GET /` already serves health |
| Use Traefik labels in local compose | Cross-stack coupling (parent network) or local Traefik (unnecessary complexity) |

## Consequences

**Positive:**
- Agent deterministic launch (explicit command, no merge side-effects)
- Production deployment unaffected (parent stack untouched)
- Port isolation (no conflict with dev/parent)
- Health probe matches existing Dockerfile + App.js

**Negative:**
- Third compose file to maintain (documentation burden)
- Operator must use explicit `-f` flag (not `docker compose up`)

**Neutral:**
- `.env.example` missing (latent docs bug) — create during execute phase
