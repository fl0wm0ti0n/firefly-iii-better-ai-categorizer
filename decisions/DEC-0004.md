# Decision DEC-0004

Status: Accepted

## Context

Configuration (mappings, auto-cat rules, failed transactions, extraction config)
persists as JSON files under `data/`. Alternative: SQLite, PostgreSQL, or Firefly
III itself as config store.

## Options

1. **JSON files in `data/`** — current approach via `storage.js`
2. **SQLite** — single-file DB, better query/concurrency
3. **Store rules in Firefly III** — tags, notes, or custom fields

## Recommendation

**Option 1: Retain JSON files.**

Matches single-user sidecar deployment. Docker volume mount on `data/` is
simple. File count is small (<10 files). No migration needed for existing installs.

## Impact

- **Positive:** Zero infra overhead; human-readable config; easy backup/restore.
- **Negative:** No concurrent write safety (acceptable at concurrency 1); corruption risk R8 mitigated by backups.
- **Linked:** architecture Components §5, codebase-map Service layer table.
