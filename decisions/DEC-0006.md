# Decision DEC-0006

Status: Accepted

## Context

US-0001 requires automated tests for `#resolveCategory` precedence on monolithic
`App.js` (~3.3k LOC). PO discovery proposed three test-seam options:

- **A:** Minimal injectable-deps factory (`App.createForTest(deps)`)
- **B:** Extract pipeline to `src/CategorizationPipeline.js`
- **C:** HTTP integration tests (scope_out)

Research R-0008 validated `node:test` + ESM import patterns and ran POC proving
Option A factory injection with `t.mock.fn()` stubs.

## Options

1. **Option A — injectable-deps factory** — add `static createForTest(deps)` and
   `resolveCategoryForTest()` wrapper on `App`; inject plain-object service stubs.
2. **Option B — pipeline extraction** — move `#resolveCategory` logic to a new
   module with public `resolveCategory()`; update all `App.js` call sites.
3. **Option C — HTTP integration** — test via Express routes with supertest or
   live stack (rejected in scope_out).

## Recommendation

**Option 1 (Option A)** for US-0001.

Smallest diff (~15–25 LOC) meets AC-2 without triggering scope_out App.js
decomposition. Private fields are assignable from `static createForTest` inside the
class body. Stub injection avoids `mock.module` experimental flags and async
constructor races from real mapping services.

Option B remains a valid follow-up when monolith split is scheduled; not required
for precedence regression coverage.

## Impact

- **Positive:** Unblocks US-0003/US-0004 regression tests; zero new npm deps;
  `node --test tests/` runner works on Node 18.
- **Negative:** Narrow test-only API on production `App` class; Option B debt
  remains if long-term testability without `App` coupling is desired.
- **Linked:** R-0008, US-0001, DEC-0001, DEC-0003.
