# DEC-0015: Unified Categorization Panel (Panel Merge Strategy)

## Status
Accepted

## Context
US-0005 requires consolidating the admin UI. Currently, the sidebar Categorizer group
exposes three separate entries for bulk operations: "Bulk Categorization" (panel-manual),
"Batch Jobs" (panel-batch), and "Individual Jobs" (panel-individual). This forces operators
to navigate between panels to start a job and monitor progress.

Additionally, "Test Webhook" lives in the Maintenance group, creating workflow fragmentation.

## Decision
Create a single unified `panel-categorization` that replaces panel-manual, panel-batch, and
panel-individual. Move the Test Webhook form into this unified panel.

### Panel Structure
```
panel-categorization
├── Scope selector (segmented control: Withdrawals / Deposits / Both)
├── Run actions
│   ├── Process Uncategorized button
│   ├── Process All button
│   └── Test Webhook button (with collapsible form)
├── Progress display (shared)
└── Job monitor
    ├── Batch jobs sub-container (id="batch-mount")
    └── Individual jobs sub-container (id="mount")
```

### Sidebar Impact
- Categorizer group: 10 entries → 8 entries
- Remove: panel-manual, panel-batch, panel-individual, panel-webhook (from Maintenance)
- Add: panel-categorization
- Preserve: panel-reviews (US-0004) as separate entry

### DOM Mount Points
Keep `mount` and `batch-mount` IDs unchanged. Socket.io handlers already route to correct
containers — no handler changes needed.

## Consequences

### Positive
- Single workflow: start job → monitor progress → view results without panel switching
- Reduced cognitive load: one entry point for categorization operations
- AC-1 satisfied: sidebar consolidation complete
- AC-2 satisfied: all run actions in one panel
- AC-4 satisfied: integrated job monitor with live updates
- AC-5 satisfied: operator stays on unified panel after starting operations

### Negative
- Large monolithic HTML refactor (~5.9k LOC, no UI tests)
- Risk of regression in batch controls if DOM structure changes incorrectly
- Mitigation: preserve render functions unchanged; only move DOM containers

### Neutral
- Test Webhook moves from Maintenance to Categorizer (logical grouping)
- Auto-jump code (lines 1551, 1573) removed — no longer needed
- Stale copy (line 1617) updated to reference "job monitor below"

## Alternatives Considered

### Alternative 1: Keep Separate Panels with Cross-Links
**Description:** Maintain panel-manual, panel-batch, panel-individual as separate entries
but add cross-links between them.

**Rejected because:**
- Does not reduce sidebar clutter (10 entries remain)
- Still requires panel switching for run → monitor workflow
- AC-1 not satisfied

### Alternative 2: Tabbed Interface Within Unified Panel
**Description:** Single panel with tabs for "Run", "Batch Jobs", "Individual Jobs".

**Rejected because:**
- Adds UI complexity (tab state management)
- Operator must still switch tabs to see progress
- Violates AC-5 (operator stays on unified panel)

### Alternative 3: Expandable Sections Within Unified Panel
**Description:** Single panel with collapsible sections for each sub-feature.

**Rejected because:**
- Similar to chosen approach but with more visual complexity
- Segmented scope control + inline job monitor is simpler
