# DEC-0019 — Extend dev-environment.json schema_version 1 with AC-1 fields

## Status

Accepted (2026-06-22)

## Context

US-0006 AC-1 requires `.cursor/dev-environment.json` to declare deterministic local launch configuration: `start_command`, `health_url`, `poll_seconds`/`poll_interval_seconds`, `env_file`, `required_env_vars`, `browser_probe_url`.

The existing `.cursor/dev-environment.json.example` uses `schema_version: 1` but lacks these fields. Research R-0023 Q2 identified the decision: bump to `schema_version: 2` or keep `1` and add fields as extensions?

## Decision

**Keep `schema_version: 1`** and add AC-1 fields as additive extensions.

**Rationale:**

1. **No existing consumer** — `.cursor/dev-environment.json` is not yet generated; only `.example` exists. No backward compatibility concern.
2. **Additive fields** — new fields (`start_command`, `health_url`, `poll_seconds`, etc.) are optional extensions. Schema `1` consumers (if any) ignore unknown fields.
3. **Avoid unnecessary version bump** — `schema_version: 2` signals breaking change. US-0006 is additive, not breaking.
4. **Future-proofing** — if incompatible changes arrive later (e.g., restructure `connect` object, rename `health_url`), bump to `schema_version: 2` at that time.

## Schema extension

Add fields to existing `schema_version: 1`:

```json
{
  "schema_version": 1,
  "detected_mode": "docker-host-local",
  "operator_seeded": true,
  "compose_file": "docker-compose.local.yml",
  "service": "firefly-ai-categorizer",
  "start_command": "docker compose -f docker-compose.local.yml up -d",
  "stop_command": "docker compose -f docker-compose.local.yml down",
  "health_url": "http://localhost:3001/",
  "poll_seconds": 60,
  "poll_interval_seconds": 2,
  "env_file": ".env",
  "required_env_vars": ["FIREFLY_URL", "FIREFLY_PERSONAL_TOKEN", "OPENAI_API_KEY"],
  "browser_probe_url": "http://localhost:3001/",
  "connect": {
    "endpoint": "http://localhost:3001",
    "health_path": "/",
    "hostEnv": "DEV_HOST",
    "portEnv": "DEV_PORT",
    "protocolEnv": "DEV_PROTOCOL"
  },
  "rebuild_recipe": {
    "default_tier": "B",
    "restart_on_source_change": false
  },
  "env_refs": ["DEV_HOST", "DEV_PORT", "DEV_PROTOCOL"],
  "evidence_refs": ["docs/engineering/research.md#r-0023"]
}
```

**Key fields per AC-1:**

| Field | Purpose | Value |
|-------|---------|-------|
| `start_command` | Deterministic agent trigger | `docker compose -f docker-compose.local.yml up -d` |
| `stop_command` | Deterministic agent teardown | `docker compose -f docker-compose.local.yml down` |
| `health_url` | Agent health poll target | `http://localhost:3001/` |
| `poll_seconds` | Max poll duration | `60` (matches scratchpad `UAT_PROCESS_HEALTH_POLL_SECONDS`) |
| `poll_interval_seconds` | Poll interval | `2` (matches scratchpad `UAT_PROCESS_HEALTH_POLL_INTERVAL_SECONDS`) |
| `env_file` | Env file path | `.env` |
| `required_env_vars` | Required env var names | `["FIREFLY_URL", "FIREFLY_PERSONAL_TOKEN", "OPENAI_API_KEY"]` |
| `browser_probe_url` | Cursor browser MCP target | `http://localhost:3001/` |
| `compose_file` | Identify compose target | `docker-compose.local.yml` |
| `service` | Identify service name | `firefly-ai-categorizer` |
| `connect.health_path` | Health endpoint path | `"/"` (not `"/health"`) |

## Alternatives considered

| Alternative | Rejected because |
|-------------|------------------|
| Bump to `schema_version: 2` | No existing consumer; additive fields don't break compatibility |
| Create new `.cursor/dev-environment-uat.json` | Fragmentation; single source of truth preferred |
| Omit AC-1 fields from example | `.example` should document all supported fields |

## Consequences

**Positive:**
- Single schema version (`1`) — no migration complexity
- Additive fields — future consumers can ignore unknown fields
- Complete AC-1 coverage — all required fields present
- `.example` serves as documentation

**Negative:**
- Schema grows over time (field sprawl)
- No version boundary signals "this is the US-0006 shape"

**Neutral:**
- `.cursor/dev-environment.json` (real file, not `.example`) generated at execute phase
