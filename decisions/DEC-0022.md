# DEC-0022 — Pipeline placement at AI-hint slot

- **Status**: Accepted
- **Date**: 2026-06-28
- **Phase**: architecture (US-0007)
- **Work items**: US-0007

## Context

US-0007 adds a direct-assign mode to keyword mappings. The question is where in `#resolveCategory()` to insert the direct-assign check.

Current pipeline precedence (line 1180–1253 of `src/App.js`):

1. **Step 1 — Account → Category Mapping** (line 1187–1200): highest tier, returns immediately if matched
2. **Step 2 — Auto-categorization** (line 1202–1210): foreign/travel/auto rules
3. **Step 3 — AI-hint slot** (line 1212–1222): keyword mapping AI hint (description hint for OpenAI)
4. **Step 4 — History analysis** (line 1224–1243): US-0004 history dominance check
5. **Step 5 — OpenAI classify** (line 1245–1253): AI call
6. **Step 6 — History-vs-AI comparison** (line 1256+): US-0004 review queue logic

Options considered:

### Option A: Before account mapping (line 1187)

- Insert direct-assign check before step 1.
- Direct-assign gets highest precedence.
- **Pros**: none significant.
- **Cons**: violates user's explicit intake decision (option c); breaks account mapping's established highest-tier status (DEC-0001).
- **Rejected**.

### Option B: Before auto-categorization (line 1202)

- Insert between step 1 and step 2.
- Direct-assign gets second-highest precedence (after account mapping).
- **Pros**: none significant.
- **Cons**: violates user's explicit intake decision (option c); skips auto-categorization (foreign/travel rules), which operators rely on for deterministic categorization without AI.
- **Rejected**.

### Option C: At AI-hint slot (line 1214) — chosen

- Insert direct-assign check at the existing AI-hint slot.
- Replaces logic at that position, not position itself.
- Precedence: account mapping → auto-cat → [direct assign OR AI hint] → history → OpenAI.
- **Pros**:
  - Matches user's explicit intake decision (option c, confirmed 2026-06-27).
  - Preserves account mapping + auto-categorization precedence (DEC-0001, DEC-0011).
  - Minimal code change — replaces existing AI-hint call with a conditional check.
  - Semantically coherent: both direct-assign and AI-hint are keyword-mapping outcomes; one assigns directly, the other hints.
- **Cons**:
  - None significant. The slot is semantically appropriate for keyword-mapping outcomes.

### Option D: After history analysis, before OpenAI (line 1243)

- Insert between step 4 and step 5.
- Direct-assign gets lower precedence than history analysis.
- **Pros**: none significant.
- **Cons**: violates user's explicit intake decision (option c); history analysis (US-0004) is meant to compare against AI, not against direct-assign; direct-assign is authoritative, not a "suggestion" to be compared.
- **Rejected**.

### Option E: After OpenAI (line 1253)

- Insert after step 5.
- Direct-assign gets lowest precedence.
- **Pros**: none.
- **Cons**: defeats the entire purpose of direct-assign (bypass OpenAI); violates user's explicit intake decision (option c).
- **Rejected**.

## Decision

**Option C** — insert direct-assign check at the AI-hint slot (line 1214).

Final pipeline order after US-0007:

1. Step 1: Account → Category Mapping
2. Step 2: Auto-categorization
3. Step 3: [Direct Keyword Assign OR AI Hint] — direct-assign check first, then AI-hint fall-through
4. Step 4: History analysis (unreachable on direct-assign)
5. Step 5: OpenAI classify (unreachable on direct-assign)
6. Step 6: History-vs-AI comparison (unreachable on direct-assign)

## Justification

- User confirmed option (c) at intake (2026-06-27, `handoffs/po_to_tl.md` US-0007 section).
- Preserves existing precedence guarantees (DEC-0001, DEC-0011–DEC-0014).
- Semantically coherent: both direct-assign and AI-hint are keyword-mapping outcomes.
- Minimal code change.

## Fall-through rule on unknown target category

When `directAssignment.assigned === true` but the target category does not exist in the Firefly categories map (`categories.has(directAssignment.category) === false`), the pipeline falls through to the AI-hint path.

**Rationale**: this mirrors the existing account-mapping miss semantics at line 1197–1200 (warn + skip, fall through to next tier). A direct-assign match with an invalid target is a configuration error, not a hard failure; falling through allows the AI to still attempt categorization.

This rule is implemented in `src/App.js` `#resolveCategory()` as a conditional check before the early return.

## Risks

1. **Operator expects direct-assign to be authoritative regardless of target validity**: mitigated by warning log when fall-through occurs, plus UI validation that category exists in dropdown (already present in `mappingTargetCategory` select).
2. **History analysis becomes unreachable on direct-assign**: intended behavior — direct-assign is authoritative, no comparison needed. Same policy as account mapping (step 1 also skips history analysis).

## Alternatives rejected

- Option A (before account mapping): violates user decision, breaks DEC-0001.
- Option B (before auto-categorization): violates user decision, skips auto-categorization.
- Option D (after history analysis): violates user decision, misinterprets direct-assign semantics.
- Option E (after OpenAI): defeats purpose, violates user decision.

## Closure criteria

- AC-4: direct-assign at AI-hint slot, after account mapping + auto-cat (**covered by Option C**)
