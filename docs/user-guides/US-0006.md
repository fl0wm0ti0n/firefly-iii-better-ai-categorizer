# US-0006: Agent-Driven Local Categorizer Launch

## Purpose

US-0006 enables the Cursor IDE agent (and operators) to launch the Firefly III AI Categorizer locally on a deterministic port, poll its health endpoint, and run browser UAT probes via the Cursor browser MCP. This supports automated testing and development without affecting the production deployment at `categorizer.omniflow.cc`.

## Prerequisites

1. **Docker and Docker Compose installed**
   - Docker Engine 20.10+ or Docker Desktop
   - Docker Compose v2+ (the `docker compose` subcommand)

2. **Port 3001 available**
   - The local service binds to host port 3001
   - Verify with: `lsof -i :3001` (should return nothing)

3. **Environment file configured**
   - Copy the example: `cp .env.example .env`
   - Set required variables:
     - `FIREFLY_URL` — your Firefly III instance URL
     - `FIREFLY_PERSONAL_TOKEN` — API token from Firefly III
     - `OPENAI_API_KEY` — OpenAI API key for AI classification

4. **Source code available**
   - Clone or download the `firefly-iii-ai-categorize` repository

## Usage Steps

### 1. Launch the Local Service

Run the dev-launch script from the repository root:

```bash
bash scripts/dev-launch.sh
```

The script will:
- Check if port 3001 is available
- Verify `.env` exists
- Build the Docker image (first run ~2-3 minutes)
- Start the container
- Poll `http://localhost:3001/` every 2 seconds
- Exit when HTTP 200 is received (max 60 seconds)

### 2. Verify the Service is Running

Once the script reports success:

```bash
curl http://localhost:3001/
# Should return: "OK"
```

Open the admin UI in your browser:
```
http://localhost:3001/
```

### 3. Browser UAT Probe (Agent Workflow)

The Cursor IDE agent can automate browser-based testing:

**Agent workflow:**
1. Read `.cursor/dev-environment.json` for launch configuration
2. Execute `start_command`: `docker compose -f docker-compose.local.yml up -d`
3. Poll `health_url`: `http://localhost:3001/` until HTTP 200 (max 60s)
4. Use browser MCP to navigate to `browser_probe_url` (same URL)
5. Collect console errors and API responses via CDP
6. Execute `stop_command`: `docker compose -f docker-compose.local.yml down`

**Manual browser UAT probe:**
1. Open `http://localhost:3001/` in browser
2. Open DevTools (F12) → Console
3. Check for errors (expected: none on fresh launch)
4. In Console, run: `fetch('/api/reviews').then(r => r.json()).then(console.log)`
5. Expected: `{ success: true, reviews: [...] }` or `{ success: false, error: "..." }`
6. **Do NOT see**: HTML response causing `SyntaxError: Unexpected token '<'`

### 4. Stop the Service

```bash
bash scripts/dev-launch.sh --stop
```

Or directly:
```bash
docker compose -f docker-compose.local.yml down
```

## Example

**Complete local UAT workflow:**

```bash
# 1. Configure environment
cp .env.example .env
nano .env  # Fill in FIREFLY_URL, FIREFLY_PERSONAL_TOKEN, OPENAI_API_KEY

# 2. Launch and wait for health
bash scripts/dev-launch.sh
# Output: "Service healthy after 12s (HTTP 200)."
#         "Browse to: http://localhost:3001/"

# 3. Run tests (manual or automated)
curl http://localhost:3001/
# "OK"

curl http://localhost:3001/api/reviews
# {"success":true,"reviews":[...]}

# 4. Stop when done
bash scripts/dev-launch.sh --stop
# "Service stopped."
```

**Agent-driven workflow (from `.cursor/dev-environment.json`):**

```bash
# Agent reads config
cat .cursor/dev-environment.json | jq -r '.start_command'
# "docker compose -f docker-compose.local.yml up -d"

# Agent executes start_command
docker compose -f docker-compose.local.yml up -d

# Agent polls health_url (pseudo-code)
for i in {1..30}; do
  if curl -sf http://localhost:3001/ > /dev/null; then
    echo "Healthy"
    break
  fi
  sleep 2
done

# Agent runs browser probe via MCP (see docs/engineering/browser-uc.md)
# Agent executes stop_command
docker compose -f docker-compose.local.yml down
```

## Limitations

1. **Ephemeral — no persistence guarantee**
   - Container restarts lose in-memory state
   - Data directory (`./data`) persists across restarts but is not backed up
   - Use for testing/development, not production data

2. **Single instance per host**
   - Port 3001 binds to localhost — only one instance per machine
   - Multiple developers on the same host need different ports (out of scope)

3. **No HTTPS/TLS**
   - Local service is HTTP only
   - Firefly III personal token transmitted in plaintext (acceptable for localhost)
   - Never expose port 3001 to the internet

4. **Browser MCP localhost access**
   - Cursor browser MCP can reach `localhost:3001` (confirmed in R-0023 Q6)
   - External browsers may require firewall rules (not tested)

5. **Health probe semantics**
   - `GET /` returns `OK` (200) — matches Dockerfile HEALTHCHECK
   - `/health` endpoint does not exist in this repo (latent docs bug per R-0023 Q1)
   - Future: optionally add `/health` alias for consistency

6. **Production deployment unaffected**
   - Local compose is separate from production (`categorizer.omniflow.cc`)
   - No shared state, no cross-contamination
   - Per DEC-0018: explicit `-f` flag prevents auto-merge

## Troubleshooting

### Port 3001 already in use

**Symptom:** `ERROR: Port 3001 is already in use.`

**Solution:**
```bash
# Find what's using port 3001
lsof -i :3001
# or
ss -tlnp | grep :3001

# Stop the other service, or stop the local categorizer:
bash scripts/dev-launch.sh --stop

# Or change the port in docker-compose.local.yml (not recommended — breaks agent contract)
```

### .env file not found

**Symptom:** `ERROR: .env file not found.`

**Solution:**
```bash
cp .env.example .env
# Edit .env and set required variables:
#   FIREFLY_URL=https://your-firefly-instance.com
#   FIREFLY_PERSONAL_TOKEN=your-api-token
#   OPENAI_API_KEY=sk-your-openai-key
```

### Image build failure

**Symptom:** Docker build fails during `npm install` or `npm ci`

**Common causes:**
- Network issue downloading npm packages
- Outdated base image (`node:18-alpine`)
- Missing `package-lock.json`

**Solution:**
```bash
# Force rebuild without cache
docker compose -f docker-compose.local.yml build --no-cache

# Or manually build to see verbose output
docker build -t firefly-ai-categorizer-local .

# Check logs for specific failures
docker compose -f docker-compose.local.yml logs
```

### Health poll timeout

**Symptom:** `ERROR: Service did not become healthy within 60s.`

**Diagnosis:**
```bash
# Check container status
docker compose -f docker-compose.local.yml ps

# Check container logs
docker compose -f docker-compose.local.yml logs firefly-ai-categorizer

# Manually test health endpoint
curl -v http://localhost:3001/
```

**Common causes:**
- `.env` missing required variables (service crashes on startup)
- Firefly III instance unreachable (service fails to connect)
- OpenAI API key invalid (service fails to authenticate)
- Port conflict or permission denied (rare)

**Solution:**
```bash
# Check .env has all required variables
grep -E '^(FIREFLY_URL|FIREFLY_PERSONAL_TOKEN|OPENAI_API_KEY)=' .env

# Restart the service after fixing .env
bash scripts/dev-launch.sh --stop
bash scripts/dev-launch.sh
```

### Browser MCP cannot reach localhost:3001

**Symptom:** Cursor browser MCP fails to navigate to `http://localhost:3001/`

**Diagnosis:**
- Verify service is running: `curl http://localhost:3001/`
- Check browser MCP permissions (see `docs/engineering/browser-uc.md`)
- Confirm browser MCP version supports localhost access (R-0023 Q6 confirmed)

**Solution:**
```bash
# Restart browser MCP server (in Cursor IDE)
# Help → Command Palette → "Browser MCP: Restart Server"

# Or restart the categorizer service
bash scripts/dev-launch.sh --stop
bash scripts/dev-launch.sh
```

### Container keeps restarting

**Symptom:** `docker compose ps` shows container restarting every few seconds

**Diagnosis:**
```bash
docker compose -f docker-compose.local.yml logs --follow
```

**Common causes:**
- Application crash (uncaught exception in Node.js)
- Missing runtime dependencies
- Insufficient memory (OOM killed)

**Solution:**
```bash
# Check logs for error
docker compose -f docker-compose.local.yml logs firefly-ai-categorizer | tail -50

# Increase memory limit (edit docker-compose.local.yml)
# Under deploy.resources.limits, increase memory: 512M → 1G

# Disable restart policy temporarily for debugging
# Change "restart: 'no'" to "restart: 'unless-stopped'" and remove restart in dev-launch.sh
```
