# US-0002 — Operator documentation map

## Purpose

This guide is a **navigation map** for Firefly III AI Categorizer documentation — not a
feature manual. It answers “where do I look?” for self-hosted operators configuring
mappings, running bulk jobs, and troubleshooting deployments. For feature descriptions
and the canonical pipeline diagram, use [README.md](../../its_magic/README.md).

## Prerequisites

- A running [Firefly III](https://www.firefly-iii.org/) instance with a Personal Access Token (PAT)
- The categorizer deployed (Docker recommended) with `FIREFLY_URL`, `FIREFLY_PERSONAL_TOKEN`, and `OPENAI_API_KEY` configured
- See [README § Installation](../../its_magic/README.md#installation) for env vars and webhook setup

## Usage steps

1. **Identify your task** — setup, pipeline order, bulk run, mapping rules, UI tool, Docker, duplicates, maintainer deep-dive, or quality gates.
2. **Open the linked canonical surface** from the table below (prefer README and legacy `docs/*_GUIDE.md` over this map).
3. **Cross-check gates** — when changing behavior or releasing, verify `TEST_COMMAND` and README coverage validators in the [runbook](../engineering/runbook.md) and [developer README](../developer/README.md).

## Limitations

- This map does **not** replace [README.md](../../its_magic/README.md) feature prose or the pipeline ASCII diagram.
- Legacy feature guides under `docs/*_GUIDE.md` may lag README; prefer README for pipeline order (DEC-0001).
- English only; no translations in this slice.
- Per-story how-tos at `docs/user-guides/US-xxxx.md` are added at execute time for other stories.

## Troubleshooting

- **Doc vs behavior mismatch** — compare symptoms with [README § Troubleshooting](../../its_magic/README.md#-troubleshooting) and the relevant legacy guide from the table above.
- **Stale architecture claims** — cross-check [CODEBASE_ANALYSIS.md](../CODEBASE_ANALYSIS.md) against README and `#resolveCategory` in `src/App.js`.
- **Validator or test failures** — see [runbook.md](../engineering/runbook.md) (`TEST_COMMAND`, `validate_readme_feature_coverage.py`) and [developer README § Quality gates](../developer/README.md#quality-gates).
